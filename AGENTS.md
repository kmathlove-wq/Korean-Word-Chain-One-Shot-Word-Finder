# 끝말잇기 한방단어 검색기 — AGENTS.md

## 프로젝트 개요

이 저장소를 수정하는 코딩 에이전트를 위한 작업 지침이다. 국립국어원 공식 Open API를 이용해 시작 단어와 끝말잇기 한방단어를 제공하는 Flask 웹앱이며, 정확한 판정·API 호출 절약·비밀 키 보호를 최우선으로 한다.

프런트엔드는 HTML/CSS/JavaScript, 백엔드는 Python/Flask/requests로 구성된다. Python 서버가 필요하므로 VS Code Live Server나 GitHub Pages만으로는 정상 동작하지 않는다.

## 기술 구성

- 백엔드: Python 3.10+, Flask, requests, python-dotenv
- 프런트엔드: Jinja HTML, CSS, 바닐라 JavaScript
- 외부 데이터: 표준국어대사전 및 우리말샘 공식 Open API
- 테스트: Python `unittest`, `compileall`, Node 구문 검사

## 파일 구조

```text
app.py                 Flask 앱과 전체 도메인 로직
templates/index.html   검색 폼과 결과 UI
static/style.css       반응형·접근성 스타일
static/main.js         fetch, 상태, 정렬, 카드 렌더링
tests/test_app.py      백엔드 자동 테스트
requirements.txt       Python 의존성
.env.example           공개 가능한 환경 변수 이름
.env                   로컬 전용 비밀 키, Git 제외
README.md              사용자 설치·운영 문서
CLAUDE.md              상세 프로젝트 지식
```

## 필수 보안 규칙

- `.env`와 API 키를 읽어서 출력하거나 답변에 포함하지 않는다.
- 키 설정 확인은 존재 여부 또는 불리언만 출력한다.
- `.env.example`에는 실제 키를 절대 넣지 않는다.
- `.env`를 Git에 추가하거나 `.gitignore` 예외로 만들지 않는다.
- 사용자가 키를 `.env.example`에 넣었다면 값을 출력하지 않고 `.env`로 옮긴 후 예시 파일을 즉시 플레이스홀더로 복원한다.
- API URL을 로그로 출력할 때 쿼리 문자열의 `key`가 노출되지 않게 한다.

## 환경 변수와 보안

```env
STDICT_API_KEY=표준국어대사전_키
OPENDICT_API_KEY=우리말샘_키
```

- 실제 키는 `.env`에만 저장하고 배포 환경에서는 같은 이름의 Secret으로 등록한다.
- `.env.example`에는 변수 이름과 플레이스홀더만 둔다.
- 환경 변수 변경 후 Flask 서버를 재시작한다.

## API 라우트

| 라우트 | 역할 |
|---|---|
| `GET /` | Jinja 템플릿으로 메인 화면 렌더링 |
| `GET /api/health` | 서버 상태와 사전별 키 설정 여부 반환 |
| `GET /api/search` | 시작 단어 검색, 필터, 한방 판정, 페이지 응답 |

`dictionary`은 `stdict`, `opendict`, `mode`는 `all`, `words`, `one-shot`을 허용한다. 검색 필터는 `Filters` 데이터 클래스와 쿼리 매개변수 이름을 일치시킨다.

## 실행과 진단

- 이 프로젝트는 Live Server로 실행하지 않는다. `python app.py`로 Flask를 실행한다.
- 기본 주소는 `http://127.0.0.1:5000`이다.
- 서버 시작 후 `/api/health`가 HTTP 200인지 확인한다.
- 환경 변수 변경 후에는 Flask 프로세스를 재시작한다.
- 장시간 실행 프로세스는 세션 ID를 보존하고, 재시작 전 기존 프로세스에 `Ctrl+C`를 보낸다.
- 사용자가 “아무 변화가 없다”고 하면 서버 로그에서 `/api/search` 도착 여부, 브라우저 캐시, 로딩 `hidden` 상태, API 응답 시간을 순서대로 확인한다.

## 코드 책임과 불변 조건

### `app.py`

- `DICTIONARIES`: 사전 이름, 공식 엔드포인트, 키 환경 변수, 상세 링크 형식.
- `TTLCache`: 같은 사전·검색어·필터·페이지 호출을 30분간 재사용.
- `Filters`: 모든 검색 필터와 캐시 키 직렬화.
- `fetch_dictionary()`: 인증, 재시도, 제한 시간, JSON/XML 파싱, 필터 적용.
- `paged_search()`: 필터를 통과한 화면 페이지 수집.
- `continuation_count()`: 후속 단어 존재 확인. 한 항목만 조회하도록 축소하면 한 글자 필터 때문에 거짓 한방 판정이 재발한다.
- `search()`: 페이지 후보의 고유 마지막 음절을 최대 8개 작업자로 병렬 분석.

### `static/main.js`

- 모든 API/사용자 문자열은 DOM 삽입 전 `escapeHtml()`을 적용한다.
- `state.params`를 이용한 다음 페이지 요청을 유지한다.
- 오류가 발생해도 `finally`에서 로딩을 숨긴다.
- 검색 요청 중 기존 결과를 잘못 누적하지 않는다.
- 정렬은 `state.words` 복사본에 적용해 원본을 변형하지 않는다.

### `static/style.css`

- `[hidden]{display:none!important}`를 제거하지 않는다. 제거하면 오류 메시지와 로딩 표시가 동시에 남을 수 있다.
- 색상만으로 한방단어를 구별하지 말고 `한방단어` 텍스트 배지를 유지한다.
- 모바일 600px, 태블릿 850px 레이아웃을 점검한다.
- 키보드 포커스와 `prefers-reduced-motion` 처리를 보존한다.

## 검색 및 판정 규칙

- 입력은 공백 제거 후 완성형 한글 `[가-힣]` 1~20자만 허용한다.
- 시작 일치 검색은 `type_search=search`, `method=start`를 사용한다.
- JSON을 우선 사용하고 XML은 안전한 대체 파서로 처리한다.
- 단어의 마지막 글자는 문자열 끝이 아니라 마지막 유효 한글 음절이다.
- 두음법칙은 `DUEUM_MAP`에서 중앙 관리하며 JavaScript에 중복 구현하지 않는다.
- 원음 또는 두음 변환음 중 하나에라도 허용 단어가 있으면 한방단어가 아니다.
- 후속 검색은 최소 100개 묶음에서 필터 통과 항목을 확인한다. 성능 최적화 시 이 정확성 조건을 깨지 않는다.
- 동일한 마지막 음절은 요청 하나의 결과를 공유한다.
- 사전 API 실패는 사용자에게 경고 메시지로 전달한다.

## 성능 규칙

- 화면에는 24개 단위로 제공한다.
- 첫 요청에서 전체 후보를 무제한 수집하거나 모든 뜻을 순차 조회하지 않는다.
- 마지막 음절별 조회는 캐시하고 독립 요청은 제한된 작업자 수로 병렬화한다.
- 공식 API에 과도한 동시 요청을 보내지 않는다. 현재 상한은 8개 작업자다.
- 제한 시간과 재시도를 없애지 않는다. 현재 연결 10초, 응답 20초, 총 2회 시도다.
- 성능 변경 후 흔한 글자 `기`와 드문 글자 `슘` 양쪽의 응답 시간과 판정을 확인한다.

## 변경 절차

1. 작업 전 `git status --short`로 사용자 변경을 확인한다.
2. 관련 파일과 테스트를 읽고 기존 동작을 보존한다.
3. 비밀 키나 실제 API 응답 전체를 출력하지 않는다.
4. 로직 변경에는 재현 가능한 회귀 테스트를 추가한다.
5. 아래 기본 검사를 실행한다.
6. 실제 API 검사는 키가 설정된 경우 최소 호출로 수행한다.
7. 변경 파일, 검증 결과, 남은 제한을 사용자에게 보고한다.

## 기본 검증 명령

```bash
python -m unittest discover -s tests -v
python -m compileall -q app.py tests
node --check static/main.js
git diff --check
```

API 키를 노출하지 않는 상태 검사:

```bash
curl -sS http://127.0.0.1:5000/api/health
```

## 회귀 테스트 필수 사례

- `validate_query()`: 빈 값, 영문, 숫자, 특수문자, 20자 초과 거부.
- `last_hangul_syllable()`: 괄호·숫자·하이픈 뒤에서도 마지막 한글 반환.
- `get_dueum_variants()`: 원음과 변환음의 중복 없는 반환.
- JSON과 XML 응답에서 단어·품사·뜻·코드 정규화.
- 첫 후속 결과가 필터로 제외되더라도 뒤에 허용 단어가 있으면 한방이 아님.
- `/api/health`, `/api/search` 성공 및 400/502 오류 계약.
- 같은 표제어의 중복 병합.

## 문서와 배포

- 동작, 환경 변수, 실행 명령이 바뀌면 README와 CLAUDE.md도 갱신한다.
- `AGENTS.md`와 `CLAUDE.md`는 각각 200줄 이하를 유지한다.
- GitHub Pages 단독 배포를 제안하지 않는다. Flask 지원 호스팅과 GitHub 저장소를 연결한다.
- 운영 배포에는 WSGI 서버, HTTPS, 환경 변수 Secret, 요청 제한, 공유 캐시가 필요하다.
- 배포 주소가 정해지면 두 국립국어원 API의 사용 URL 설정도 갱신하도록 안내한다.

## 알려진 제한

- 국립국어원 API 응답 속도와 일일 호출 제한에 영향을 받는다.
- API 분류 필드 차이로 일부 필터가 사전 웹사이트와 완전히 같지 않을 수 있다.
- 두음 변형의 이어갈 단어 수에는 중복이 포함될 수 있다.
- 한방 판정은 선택한 사전과 필터 기준이며 실제 게임별 허용 규칙과 다를 수 있다.

## 작업 원칙

- 사용자의 기존 변경을 임의로 되돌리지 않는다.
- 공식 API 대신 크롤링, 비공식 프록시, 임시 가짜 데이터를 넣지 않는다.
- API 응답 구조를 추측해 필드를 만들지 말고 공식 문서 또는 실제 응답으로 확인한다.
- 정확도를 희생하는 성능 개선은 하지 않는다. 불가피한 근사치는 UI와 문서에 명시한다.
- 요청받지 않은 커밋이나 push는 수행하지 않는다.
